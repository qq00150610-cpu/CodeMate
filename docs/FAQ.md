## 安装问题

### Q1: 提示「应用未安装」或安装失败？

**A:** 可能的原因和解决方案：

1. **签名不一致**：如果是覆盖安装，新版本的签名必须与旧版本一致
2. **存储空间不足**：确保有至少 500MB 的可用空间
3. **Android 版本不兼容**：需要 Android 7.0 (API 24) 或更高版本

```bash
# 检查设备 Android 版本
adb shell getprop ro.build.version.sdk
```

### Q2: 首次安装后无法打开？

**A:** 这是因为应用需要首次初始化。请执行以下操作：

1. 等待初始化完成（可能需要 5-10 分钟）
2. 如果长时间无响应，尝试重启设备
3. 清除应用数据后重新安装

```bash
# 清除应用数据
adb shell pm clear com.codemate.editor
```

### Q3: 如何更新应用？

**A:** 有两种方式：

1. **应用内更新**：
   - 设置 → 检查更新
   - 下载并安装新版本

2. **手动更新**：
   - 下载新的 APK 文件
   - 覆盖安装（签名必须一致）

## 使用问题

### Q4: 如何创建新项目？

**A:** 

1. 点击主界面的「新建项目」
2. 选择项目模板
3. 输入项目名称和路径
4. 点击「创建」

### Q5: 支持外部键盘吗？

**A:** 是的！应用支持外部键盘：

- 自动检测并启用键盘模式
- 隐藏软键盘快捷键栏
- 支持常用快捷键（Ctrl+C, Ctrl+V, Ctrl+Z 等）
- 部分设备可能需要重启应用

### Q6: 如何调整字体大小？

**A:** 

- **双指捏合**：在编辑器中使用捏合手势缩放
- **设置菜单**：设置 → 编辑器 → 字体大小
- **性能模式**：低内存模式下自动缩小字体

### Q7: 文件丢失怎么办？

**A:** 

1. 检查是否在正确的文件夹中
2. 使用文件管理器查看 `/storage/emulated/0/Download/CodeMate/`
3. 应用数据保存在 `/data/data/com.codemate.editor/files/`

## AI 功能问题

### Q8: AI 功能无法使用？

**A:** 请按以下步骤排查：

1. **检查网络**：确保设备已连接互联网
2. **检查后端配置**：
   - 设置 → AI 设置 → 后端地址
   - 确认地址正确且服务运行中
3. **检查 API 配置**：
   - 联系管理员确认 API Key 有效
   - 检查是否有调用配额

### Q9: AI 响应很慢？

**A:** 可能的原因：

1. **网络延迟**：切换到更快的网络
2. **服务器负载**：高峰期响应较慢
3. **请求内容过大**：尝试简化问题或减少代码量

**解决方案**：
- 使用「通义千问-超快速」模型
- 减少对话历史长度
- 简化问题描述

### Q10: 如何更改 AI 模型？

**A:** 

1. 打开 AI 侧边栏
2. 点击右上角「设置」图标
3. 选择「模型」选项
4. 选择需要的模型

可用模型：
| 模型 | 速度 | 能力 |
|------|------|------|
| 通义千问-超快速 | 最快 | 日常对话 |
| 通义千问-均衡 | 中等 | 复杂任务 |
| 通义千问-最强 | 较慢 | 深度分析 |

### Q11: 提示「验证码错误」？

**A:** 这是正常的防滥用机制：

1. 点击刷新按钮获取新验证码
2. 仔细识别图片中的字符
3. 区分大小写（如果有）

## 性能问题

### Q12: 应用很卡顿？

**A:** 尝试以下优化：

1. **开启低内存模式**：
   - 设置 → 性能 → 低内存模式

2. **关闭动画**：
   - 设置 → 性能 → 禁用动画

3. **限制历史记录**：
   - 设置 → AI → 限制历史条数

4. **重启应用**：
   - 完全关闭后重新打开

### Q13: 内存不足怎么办？

**A:** 

1. **关闭不需要的项目**
2. **清除缓存**：设置 → 存储 → 清除缓存
3. **使用低内存模式**
4. **重启设备**

### Q14: 设备发热严重？

**A:** 可能是以下原因：

1. **代码高亮渲染**：关闭语法高亮
2. **后台进程**：检查是否有后台更新
3. **长时间使用**：适当休息

## 开发问题

### Q15: 如何调试应用？

**A:** 

1. **Flutter 调试**：
   ```bash
   flutter run
   ```

2. **原生代码调试**：
   - Android Studio 打开 `android/` 目录
   - 设置断点并运行 Debug 模式

3. **WebView 调试**：
   - 设置 → 开发者选项 → 启用 WebView 调试
   - Chrome 打开 `chrome://inspect`

### Q16: 如何查看日志？

**A:** 

```bash
# Flutter 日志
flutter logs

# Android 日志
adb logcat | grep com.codemate.editor

# 代码服务器日志
adb shell "run-as com.codemate.editor cat files/.vscode/logs/*.log"
```

### Q17: 如何参与开发？

**A:** 

1. Fork 项目仓库
2. 创建功能分支
3. 编写代码并测试
4. 提交 Pull Request

详见 [CONTRIBUTING.md](./CONTRIBUTING.md)

### Q18: 如何配置本地后端？

**A:** 

1. 克隆后端仓库
2. 安装依赖：`npm install`
3. 配置环境变量：复制 `.env.example` 为 `.env`
4. 启动服务：`npm start`
5. 在应用设置中填入后端地址
