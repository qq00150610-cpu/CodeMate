# 性能调优指南

## 一、性能监控

### 1.1 实时监控

CodeMate 提供实时性能监控，显示：
- CPU 使用率
- 内存使用率
- 应用内存占用

### 1.2 性能报告

在设置页面可以生成完整的性能报告，包括：
- 当前性能指标
- 系统设置状态
- 优化建议

## 二、性能模式

### 2.1 普通模式

适用于大多数设备，默认配置：
- 完整的编辑器功能
- 动画效果启用
- 100 条历史记录
- 标准字体大小

### 2.2 低内存模式

当检测到设备内存低于 4GB 或内存使用率超过 90% 时自动启用：
- 降低编辑器渲染质量
- 禁用动画效果
- 限制历史记录为 20 条
- 减小字体大小
- 降低语法高亮更新频率

### 2.3 节能模式

适合低电量或长时间编码：
- 禁用动画
- 延长性能检查间隔
- 减少后台活动

## 三、优化建议

### 3.1 编辑器优化

1. **大文件处理**
   - 超过 1MB 的文件自动启用分块加载
   - 语法高亮延迟 300ms 后渲染

2. **减少重绘**
   - 避免频繁滚动时更新内容
   - 使用虚拟化列表

### 3.2 内存优化

1. **定期清理**
   - 定期清理 WebView 缓存
   - 限制对话历史数量

2. **低内存设备**
   - 建议使用低内存模式
   - 减少同时打开的文件数

### 3.3 网络优化

1. **离线功能**
   - 使用本地 Ollama 模型
   - 缓存常用资源

2. **WiFi 优先**
   - 设置中开启"仅 WiFi 下载"
   - 减少移动数据使用

## 四、冷启动加速

### 4.1 启动流程

```
应用启动
    │
    ├─► 显示 Splash Screen
    │
    ├─► 初始化 Flutter 环境
    │
    ├─► 启动 code-server 后台服务（异步）
    │
    ├─► 初始化 WebView
    │
    └─► 加载 WebView 内容
```

### 4.2 优化措施

1. **预热服务**：应用启动时异步启动 code-server
2. **资源缓存**：缓存解压后的资源
3. **增量更新**：只更新必要的文件

### 4.3 目标指标

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 冷启动时间 | 30s+ | < 5s |
| WebView 加载 | 10s | < 3s |
| 首屏交互 | 35s+ | < 8s |

## 五、调试工具

### 5.1 启用性能日志

在开发者选项中启用性能日志，查看：
- 方法执行时间
- 内存分配情况
- UI 渲染时间

### 5.2 内存分析

使用 Android Studio 的 Memory Profiler：
1. 连接设备
2. 打开 Profiler 窗口
3. 录制内存使用情况
