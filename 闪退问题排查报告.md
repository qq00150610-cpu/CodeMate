# CodeMate APP 闪退问题排查报告

## 问题概述
CodeMate APP 基于 VHEditor-Android 二次开发，构建 APK 后用户反馈打开即闪退。

---

## 排查结果

### 🔴 闪退原因（已修复）

#### 1. 缺少 MainActivity.kt（致命问题）✅ 已修复

**问题描述：**
AndroidManifest.xml 中配置了启动 Activity：
```xml
<activity android:name=".MainActivity" ... />
```

但 `android/app/src/main/kotlin/com/codemate/editor/` 目录下**只有**：
- `AIEngine.kt`
- `UpdateService.kt`

**缺少 MainActivity.kt**！

**影响：** Flutter 应用启动时找不到 Activity 类，直接抛出 `ActivityNotFoundException` 导致闪退。

**修复方案：** 已创建 `MainActivity.kt`，内容包含：
- 继承 `FlutterActivity`
- 在 `configureFlutterEngine()` 中初始化 Flutter Engine
- 注册 Flutter 插件 (`GeneratedPluginRegistrant`)
- 初始化 AI Engine 和 Update Service 的 MethodChannel

---

#### 2. assets 目录不完整 ✅ 已修复

**问题描述：**
pubspec.yaml 引用了不存在的资源：
```yaml
assets:
  - assets/
  - assets/bootstrap/    # ❌ 不存在
fonts:
  - asset: fonts/RobotoMono-Regular.ttf  # ❌ 不存在
```

**影响：** Flutter 构建时会尝试加载这些不存在的资源，可能导致构建失败或运行时资源加载错误。

**修复方案：**
1. 删除了对不存在 `assets/bootstrap/` 目录的引用
2. 删除了对不存在字体文件的引用
3. 创建了占位文件便于后续补充

---

#### 3. flutter_assets 引用问题 ✅ 已修复

**问题描述：**
build.gradle 中配置：
```groovy
assets.srcDirs = ["src/main/assets", "flutter_assets"]
```

`flutter_assets` 是 Flutter 编译时自动生成的目录，配置在 sourceSets 中可能导致路径解析问题。

**修复方案：** 修改为：
```groovy
assets.srcDirs = ["src/main/assets"]
```

---

### ⚠️ 需要确认的配置

#### 4. WebView 初始 URL 配置

**当前配置：**
```dart
InAppWebView(
  initialUrlRequest: URLRequest(
    url: WebUri('http://127.0.0.1:20000'),
  ),
  ...
)
```

**问题分析：**
- `http://127.0.0.1:20000` 是本地服务器地址
- 如果后端服务部署在 `http://47.92.220.102:3000`，需要确认是否正确
- 如果没有本地服务运行，WebView 会显示连接失败页面

**建议：** 根据实际部署情况调整初始 URL

---

#### 5. API 地址配置

**当前配置：**
- OpenAI API: `https://api.openai.com`
- Claude API: 使用默认配置
- Ollama: `http://localhost:11434`

**建议：** 确认 Ollama 的 baseUrl 是否需要修改为服务器地址 `http://47.92.220.102:11434`

---

### ✅ 正确的配置（无需修改）

#### AndroidManifest.xml 配置 ✅
- `android:usesCleartextTraffic="true"` ✅ 已配置
- `android:networkSecurityConfig="@xml/network_security_config"` ✅ 已配置
- 网络权限 `android.permission.INTERNET` ✅ 已配置

#### network_security_config.xml ✅
```xml
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>
```
Android 9+ 允许 HTTP 请求的配置已正确设置。

**主要闪退原因：缺少 MainActivity.kt**

这个问题已通过创建 MainActivity.kt 修复。其他问题（assets 目录、flutter_assets 引用）也已同步修复。

应用启动后还需要确认：
1. WebView 初始 URL 是否正确
2. 后端服务是否正常运行在 `http://47.92.220.102:3000`
